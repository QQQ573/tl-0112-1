# 🎓 暑期成长评语单生成器

浏览器端批量生成学员暑期成长评语单的工具。纯前端处理，数据不流出本地，高效安全。

## ✨ 功能特性

- 📊 **CSV 批量导入** - 支持列名映射，灵活适配不同格式的数据源
- 🎨 **Canvas 模板绘制** - 头像、五维雷达图、班主任寄语、二维码一体化渲染
- 📄 **多页 PDF 导出** - 一键合成多页 PDF，直接下载打印
- ⚡ **Web Worker 加速** - 超过 70 条自动启用 Worker 后台处理，不卡主线程
- 🔄 **失败重试机制** - 自动重试可恢复错误（图片加载、跨域问题）
- 📐 **双模板切换** - 竖版标准 / 横版紧凑，切换后预览全部失效重绘
- 🔒 **敏感信息保护** - 家庭住址默认打码，主管口令授权后查看明文
- 🌐 **CORS 图片代理** - 内置 Docker 镜像代理，解决跨域图片问题

## 🛠️ 技术栈

- **前端框架**: React 19 + TypeScript
- **构建工具**: Vite
- **CSV 解析**: PapaParse
- **PDF 生成**: jsPDF
- **二维码**: qrcode
- **加密**: crypto-js
- **图片代理**: Express + request
- **容器化**: Docker + Nginx

## ⚙️ 工作原理

### 处理流程

```
上传 CSV
    ↓
解析 + 数据校验 (分数范围、模板ID)
    ↓
判断记录数 > 70 ?
    ├─ 是 → Web Worker 批量处理
    └─ 否 → 主线程处理
    ↓
逐条渲染 Canvas
    ├─ 加载头像
    ├─ 绘制雷达图
    ├─ 渲染寄语文本
    └─ 生成二维码
    ↓
失败项自动重试 (最多3次)
    ↓
预览结果
    ↓
导出多页 PDF
```

### Web Worker 策略

- **阈值**: 70 条记录（可在 `src/types/index.ts` 中调整 `WORKER_THRESHOLD`）
- **并发**: 串行处理，避免内存飙升
- **重试**: 图片类失败自动重试 3 次，指数退避
- **通信**: postMessage 实时回传进度

### 失败原因分类

| 类型 | 原因 | 可重试 |
|------|------|--------|
| `cors` | 图片跨域加载失败 | 是（配合代理） |
| `image_load_failed` | 图片加载失败（网络问题） | 是 |
| `score_out_of_range` | 分数超出 0-100 范围 | 否 |
| `template_not_found` | 寄语模板 ID 不存在 | 否 |
| `unknown` | 其他未知错误 | 否 |

## 🔐 安全说明

### 敏感列打码

- 家庭住址等敏感字段默认展示打码形式
- 打码规则：前3位 + N个* + 后2位
- 输入主管口令可临时查看明文
- 口令使用 SHA-256 哈希校验，不存储明文

### 主管口令

默认口令：`admin123`

> ⚠️ 部署前请修改 `src/utils/mask.ts` 中的 `MASTER_PASSWORD_HASH`

生成新口令哈希：

```bash
node -e "console.log(require('crypto-js').SHA256('你的口令').toString())"
```

## 📊 70 条压测验收标准

### 性能基准

| 指标 | 竖版标准 | 横版紧凑 | 说明 |
|------|----------|----------|------|
| 总耗时 | ≤ 60 秒 | ≤ 55 秒 | 从点击生成到全部完成 |
| 平均单条 | ≤ 0.85 秒 | ≤ 0.78 秒 | 含图片加载+渲染 |
| 图片加载 | ≤ 300ms | ≤ 300ms | 单张头像平均 |
| Canvas 渲染 | ≤ 50ms | ≤ 45ms | 单张绘制耗时 |
| PDF 生成 | ≤ 10 秒 | ≤ 8 秒 | 70页合成耗时 |
| 内存峰值 | ≤ 500MB | ≤ 450MB | 浏览器进程占用 |

### 测试环境

- **CPU**: Intel Core i7 / 同级
- **内存**: 8GB+
- **浏览器**: Chrome 最新版
- **网络**: 图片 CDN 延迟 ≤ 50ms

### 验收条件

✅ 70 条记录全部处理完成（允许失败率 ≤ 5%）
✅ 总耗时 ≤ 60 秒（竖版）
✅ 成功生成可下载的 PDF 文件
✅ PDF 页数与成功记录数一致
✅ 失败项展示明确原因
✅ 处理过程中页面可正常交互（Worker 模式）
✅ 内存占用不超过 500MB
✅ 切换模板后所有预览重新渲染

### 基准测试方法

1. 准备 70 条标准测试数据（`samples/benchmark_70.csv`）
2. 打开 Chrome 开发者工具 → Performance 面板
3. 点击「开始生成」按钮
4. 等待全部完成，记录总耗时
5. 导出 PDF，验证页数和内容
6. 在 Memory 面板记录峰值内存

### 性能优化措施

- ✅ Web Worker 后台线程处理
- ✅ 复用同一个 Canvas 上下文
- ✅ JPEG 质量 0.9 平衡体积与质量
- ✅ 图片缓存（浏览器自动缓存）
- ✅ PDF 流式写入，避免一次性全量加载
- ✅ 失败重试指数退避

## 🐛 常见问题

### Q: 头像显示不出来，提示跨域错误？

A: 启动 docker-compose 中的图片代理服务，并在代码中配置代理地址。图片代理会中转请求并添加 CORS 头。

### Q: 生成速度很慢怎么办？

A: 
1. 检查网络速度，图片加载是主要瓶颈
2. 启用 Web Worker 模式（>70条自动启用）
3. 使用更小尺寸的头像图片
4. 尝试横版紧凑模板，渲染速度更快

### Q: PDF 下载后中文显示乱码？

A: 当前实现使用图片合成 PDF（每页一张 JPG），不存在字体问题。如果有问题请检查 Canvas 渲染。

### Q: 可以批量导出单张图片吗？

A: 目前仅支持多页 PDF 导出。如需单张图片，可在预览面板右键保存。
